1. Visión y Manifiesto

Gestión de Finanzas Personales es una aplicación multiplataforma (Web y Android) diseñada para el control exhaustivo de ingresos y gastos, priorizando el funcionamiento offline y la sincronización en la nube.

Estrategia Técnica

  • Frontend: React + TypeScript + Vite.
  • UI Framework: Ionic Framework (v8).
  • Mobile Runtime: Capacitor (APK Nativo).
  • Backend: Supabase (PostgreSQL, Auth, Storage).
  • Persistencia: LocalStorage + Cloud Sync.

2. Arquitectura de Software

El proyecto sigue una arquitectura modular basada en Componentes y Servicios, desacoplando la lógica de negocio de la interfaz de usuario.

Estructura del Proyecto (/src)

  • /components: UI reutilizable ("BudgetModal", "TransactionModal").
  • /pages: Vistas principales ("Dashboard", "Transactions").
  • /services: Comunicación con Supabase ("transactionService.ts", "ocrService.ts").
  • /i18n: Configuración multiidioma.

Patrones de Diseño

  • Service Repository Pattern: Las vistas nunca llaman a la BD directamente.
  • Cloud-First + Offline Cache: Los datos críticos residen en Supabase; preferencias en local.
  • Persistent Navigation: Arquitectura SPA con vistas persistentes (display: none) para navegación instantánea.

5. Sistema de Gestión de Cuentas Bancarias

Sistema completo de gestión de cuentas bancarias con seguridad tipo "bóveda" inspirado en estética D&D.

Funcionalidades Principales

  • Completado Wizard de Creación (4 Pasos):
    • Paso 1: Información básica (Nombre, Banco, País, Sucursal)
    • Paso 2: Código de seguridad con confirmación
    • Paso 3: Métodos de recuperación (Email/SMS)
    • Paso 4: Confirmación y creación
  • Completado Sistema de Bloqueo/Desbloqueo: Cuentas bloqueadas por defecto, requieren código de seguridad para acceder.
  • Completado Recuperación de Código: Sistema de tokens con expiración de 10 minutos.
  • Completado Generación Automática de IBAN: IBANs válidos generados según estándar europeo.
  • Completado Seguridad: Códigos hasheados con bcrypt, tokens únicos por cuenta.

Arquitectura de Seguridad

  • security_code_hash: Almacenamiento seguro con bcrypt (10 rounds).
  • failed_attempts: Contador de intentos fallidos (límite: 3).
  • is_locked: Estado de bloqueo de la cuenta.
  • recovery_tokens: Tabla separada con tokens de un solo uso y expiración.

8. Bugs Críticos Resueltos

Bug #1: Persistencia de Sesión

Severidad: Alta | Estado: RESUELTO

  • Problema: Usuarios forzadamente deslogueados tras registro.
  • Causa: Código de desarrollo que forzaba logout.
  • Solución: Deshabilitado código de logout forzado en App.tsx.

Bug #2: Token de Recuperación Doble-Validación

Severidad: Crítica | Estado: RESUELTO

  • Problema: Token marcado como "usado" durante verificación, causando fallo en paso final.
  • Causa: verifyRecoveryToken marcaba token como usado prematuramente.
  • Solución: Refactorización de recoveryService.ts:
    • verifyRecoveryToken: Solo valida, no marca como usado.
    • resetSecurityCode: Marca token como usado DESPUÉS de actualizar código.
  • Archivo: resources/frontend/ts/recoveryService.ts (Líneas 76-148)

Bug #3: Resolución de Módulos TypeScript

Severidad: Media | Estado: RESUELTO

  • Problema: Imports sin extensión .tsx fallaban en compilación.
  • Causa: tsconfig.app.json con verbatimModuleSyntax: true.
  • Solución: Añadidas extensiones .tsx explícitas en todos los imports de componentes.
