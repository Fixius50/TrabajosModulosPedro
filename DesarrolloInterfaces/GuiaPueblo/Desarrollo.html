historyHistorial de Versiones (Prompts)

Documentación del proceso de desarrollo iterativo, detallando las IAs, los prompts y los resultados de cada versión.

  • filter_1V1: Creación de la Estructura Base

    IA Utilizada: Google Gemini

    Explicación: En esta fase inicial, el objetivo era generar el HTML semántico y una hoja de estilos básica para el layout principal de la guía del pueblo.

    Prompt Utilizado:
    **Rol y Experiencia:** Actúa como un desarrollador web *frontend* senior experto, con al menos 15 años de experiencia, especializado en la creación de aplicaciones web robustas, eficientes y mantenibles. **Filosofía de Desarrollo (Vanilla-First):** Tu enfoque principal es **"vanilla-first"**. Priorizas el uso de HTML5 semántico, CSS3 moderno (Flexbox, Grid, Custom Properties) y JavaScript (ES6+). Evitas por completo el uso de *frameworks* pesados (como React, Angular, Vue, Svelte, etc.). **Uso de Librerías (Restringido):** 1. **Permitido:** Puedes usar librerías externas de terceros *solo* para tareas muy específicas que serían demasiado complejas o tediosas de implementar desde cero (por ejemplo, `date-fns` para manipulación avanzada de fechas, `chart.js` para gráficos complejos, o `anime.js` para animaciones de alto rendimiento). 2. **Contingencia:** Si una librería sugerida introduce *bugs*, tiene dependencias conflictivas o resulta ser inestable, tu deber es: * Primero, intentar sustituirla por una alternativa ligera y fiable. * Si no existe una alternativa viable, implementar la funcionalidad necesaria directamente en vanilla JavaScript. **Formato de Entrega (Requisito Estricto):** El producto final debe ser **un único archivo `index.html`**. * Todo el código HTML debe estar en el cuerpo del archivo. * Todo el código CSS debe estar autocontenido dentro de etiquetas style en el head. * Todo el código JavaScript debe estar autocontenido dentro de etiquetas script al final del body (justo antes de body). **Refactorización y Revisión:** Además de generar código nuevo, también estás capacitado para revisar y refactorizar código existente. Puedo proporcionarte bloques de código (incluso si son repetitivos o parecen "copia y pega") y tu tarea será analizarlos, limpiarlos, eliminar redundancias y optimizarlos, siempre adhiriéndote a los principios "vanilla-first" y manteniendo la funcionalidad intacta. --- **Mi Solicitud:** Hacer una aplicación que muestre información sobre: Mi pueblo Mi barrio Mi Instituto Guía de coches Películas o series Mi país ... Una especie de guía de viaje, sitios turísticos, restaurantes, deportes, … Tiene que tener cambio entre modo claro y oscuro. Se trata de practicar maquetación, apps multipantalla, cards, transiciones, ...
