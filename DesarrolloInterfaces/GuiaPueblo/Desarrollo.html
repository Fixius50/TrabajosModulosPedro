Explicación: Estructura inicial de la SPA (Single Page Application) utilizando Tailwind CSS para el diseño base y navegación por pestañas oculta con JS.

Prompt Utilizado:
**Rol y Experiencia:** Actúa como un desarrollador web *frontend* senior experto, con al menos 15 años de experiencia, especializado en la creación de aplicaciones web robustas, eficientes y mantenibles. **Filosofía de Desarrollo (Vanilla-First):** Tu enfoque principal es **"vanilla-first"**. Priorizas el uso de HTML5 semántico, CSS3 moderno (Flexbox, Grid, Custom Properties) y JavaScript (ES6+). Evitas por completo el uso de *frameworks* pesados (como React, Angular, Vue, Svelte, etc.). **Uso de Librerías (Restringido):** 1.  **Permitido:** Puedes usar librerías externas de terceros *solo* para tareas muy específicas que serían demasiado complejas o tediosas de implementar desde cero (por ejemplo, `date-fns` para manipulación avanzada de fechas, `chart.js` para gráficos complejos, o `anime.js` para animaciones de alto rendimiento). 2.  **Contingencia:** Si una librería sugerida introduce *bugs*, tiene dependencias conflictivas o resulta ser inestable, tu deber es:     * Primero, intentar sustituirla por una alternativa ligera y fiable.     * Si no existe una alternativa viable, implementar la funcionalidad necesaria directamente en vanilla JavaScript. **Formato de Entrega (Requisito Estricto):** El producto final debe ser **un único archivo `index.html`**. * Todo el código HTML debe estar en el cuerpo del archivo. * Todo el código CSS debe estar autocontenido dentro de etiquetas style en el head. * Todo el código JavaScript debe estar autocontenido dentro de etiquetas script al final del body (justo antes de body). **Refactorización y Revisión:** Además de generar código nuevo, también estás capacitado para revisar y refactorizar código existente. Puedo proporcionarte bloques de código (incluso si son repetitivos o parecen "copia y pega") y tu tarea será analizarlos, limpiarlos, eliminar redundancias y optimizarlos, siempre adhiriéndote a los principios "vanilla-first" y manteniendo la funcionalidad intacta. --- **Mi Solicitud:** Hacer una aplicación que muestre información sobre: Mi pueblo Mi barrio Mi Instituto Guía de coches Películas o series Mi país ... Una especie de guía de viaje, sitios turísticos, restaurantes, deportes, … Tiene que tener cambio entre modo claro y oscuro. Se trata de practicar maquetación, apps multipantalla, cards, transiciones, ...

Prompt Utilizado:
Actúa como un Desarrollador Frontend Senior y Diseñador UI/UX experto en CSS creativo y JavaScript Vanilla, adoptando una filosofía "Vanilla-First" pero permitiendo el uso estratégico de librerías ligeras. Tu objetivo es crear una Single Page Application (SPA) completa contenida en un único archivo index.html sobre el municipio de Valdemoro (Madrid), diseñada bajo una temática visual "Greco-Romana Peninsular". El stack tecnológico obligatorio incluye HTML5 semántico, Tailwind CSS vía CDN para la maquetación general, CSS personalizado en etiquetas style para efectos avanzados, Materialize CSS vía CDN únicamente para la funcionalidad del componente Carousel gestionando sus conflictos de estilo, JavaScript 100% Vanilla (ES6+) sin frameworks reactivos, iconos de Feather Icons y las fuentes de Google Fonts "Cinzel" para títulos y "Lato" para el cuerpo de texto. La identidad visual debe evocar la antigüedad clásica utilizando una paleta de colores específica compuesta por Oro Antiguo (#C5A059), Rojo Pompeyano (#8E352E), Azul Imperial (#2C3E50), Marfil o Mármol (#F2F0E6) y Negro para contrastes. Debes implementar texturas de fondo con ruido sutil y estilizar las tarjetas con un efecto de bloque de mármol, aplicando sombras profundas y bordes dobles dorados para simular relieves arquitectónicos. Al inicio del script, crea un objeto constante llamado APP_CONFIG para centralizar la configuración, incluyendo un placeholder para la API KEY de OMDB/TMDb y todas las URLs de las imágenes organizadas por sección, utilizando placehold.co con textos temáticos para facilitar la sustitución futura. Desarrolla un Header o encabezado con estilo de "Friso Romano" en azul oscuro con borde dorado que contenga el logo a la izquierda, un botón "VOLVER" que permanezca oculto en la página de inicio y regrese a ella desde otras secciones, un buscador global funcional que indexe las secciones internas y redirija al usuario, y botones de acción a la derecha para cambiar entre modo claro/oscuro y un botón de estrella para acceder a una sección secreta. Asegúrate de que la etiqueta nav de Materialize no inyecte sus estilos predeterminados, utilizando divs o reseteos CSS específicos para mantener la integridad del diseño de Tailwind. Para la pantalla de inicio, implementa una narrativa visual jerárquica que muestre dos "Murales" o imágenes enmarcadas como cuadros de museo representando a España y Madrid, seguidos de una tarjeta descriptiva de Valdemoro. La navegación principal debe ser un Carrusel Horizontal implementado con Materialize pero altamente personalizado mediante CSS avanzado para que las tarjetas de navegación tengan forma de Vasijas Romanas (ánforas o cráteras) utilizando border-radius asimétricos y pseudo-elementos para las asas. Este carrusel debe incluir flechas de navegación manual a los lados para desplazar los elementos paso a paso y debe reinicializarse correctamente al volver a la pantalla de inicio para evitar errores de renderizado. Construye las secciones internas transformando el concepto de listados simples en guías de servicios y entretenimiento. La pantalla de Servicios de Automoción debe funcionar como un directorio con pestañas estilo bloque de piedra, ofreciendo un buscador con filtros tipo select para gasolineras, talleres y concesionarios, además de una pestaña para enlaces a portales externos. La pantalla de Entretenimiento debe dividir su contenido en Cine y Teatro; la sección de Cine debe conectarse a la API de OMDB (con un fallback de datos locales robusto si falla) para mostrar carteleras reales y modales con detalles, mientras que la de Teatro mostrará una cartelera local ficticia. La pantalla de "Mi Pueblo" debe ser un grid de tarjetas de mármol con los servicios principales del Ayuntamiento y enlaces a su web oficial. Finalmente, implementa la pantalla secreta "Astra Valdemori", accesible desde el botón de estrella del header. Esta sección debe tener un diseño inmersivo que ocupe toda la pantalla con posicionamiento fijo, forzando un fondo negro independientemente del tema elegido. Debe incluir un GIF de fondo de una antorcha o fuego con un overlay oscuro para garantizar la legibilidad, y mostrar una leyenda épica sobre el origen mítico de Valdemoro con una tipografía cuidada. Todo el código debe ser robusto, manejando errores de carga de imágenes o APIs mediante datos de respaldo locales para asegurar que la aplicación siempre sea funcional y visualmente atractiva.
